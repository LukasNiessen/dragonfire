# Launch Roadmap

Proofstrike's source-review MVP is intended to be usable as an open-source CI/CD security gate. The roadmap below is about adoption, ecosystem growth, and commercial packaging around that core.

## Open-Source Launch

- Publish the repository with the TypeScript CLI, CI workflow, fixture app, and docs.
- Position the project as staged white-box red teaming for CI/CD.
- Publish example PRs that show tenant isolation, missing auth, webhook signature, and SQL injection findings.
- Encourage community matcher and knowledge-pack contributions.

## Community Growth

- Maintain curated built-in packs for web auth, injection, AI-app security, CI/CD, and IaC.
- Add a public issue template for new matcher requests.
- Add fixture requirements for every matcher contribution.
- Publish "how to write a matcher" and "how to write a knowledge pack" guides.

## Go-To-Market Motion

- Use source-only analysis on open-source projects and coordinated case studies.
- Offer a launch assessment for teams that want Proofstrike configured in GitHub Actions or GitLab CI.
- Package custom hotspot, policy, and knowledge files as a consulting deliverable.
- Build a hosted dashboard only after the CLI has real adoption signals.

## Commercial Expansion

- Team dashboard for multi-repo evidence history.
- Private pack registry for companies.
- Managed runner and policy governance.
- SSO/RBAC/audit logs for enterprise.
- Consulting packages for rollout, custom matchers, and remediation workflow.

## Future Product Expansion

Future grey-box and black-box testing should reuse the same stage, asset, signal, evidence, validation, and policy model. The source-review core remains the foundation.
